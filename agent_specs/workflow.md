# Thesis Revision Agent Pipeline

This directory is the provider-neutral source of truth for the blind-review thesis revision agent system.

The project is not only a Claude Code Skill. The Skill is one adapter that lets Claude Code discover and run this pipeline. Other hosts, including Codex-style agents, should read this directory and call the same deterministic scripts.

## Roles

- Orchestrator: validates inputs, creates the run directory, calls deterministic scripts, dispatches revision agents, validates outputs, and builds final reports.
- `deep-revision-planner`: writes one human-readable structured Markdown Revision Card for exactly one review comment. The card must be a complete manuscript revision dossier, not a brief suggestion.
- `revision-solution-auditor`: strictly audits one parsed revision plan JSON and decides whether it passes, needs revision, needs author input, or needs manual review. It must reject advice-only or underdeveloped plans.
- Deterministic scripts: parse DOCX/review files, build mappings, convert Markdown to JSON, validate schemas, render reports, and create Word suggestion copies.

## Runtime Contract

User-facing outputs should live under:

- `outputs/`
- `revision_plan_notes/`
- `audits/`

Internal artifacts should live under:

- `paper/`
- `assets/`
- `artifacts/`
- `revision_plans/`
- `review/`

Planner agents write Markdown first. JSON is an internal representation generated by `scripts/parse_revision_plan_markdown.py` or deterministic scaffold tools.

## Quality Contract

The system is intended to produce a near-final revised manuscript package. A valid deep result must contain:

- detailed diagnosis for every review comment;
- paper-grounded original excerpts and exact anchors;
- paste-ready replacement/addition text for every text-level action;
- synchronized changes across abstract, methods, experiments, conclusions, figures, tables, references, or innovation points when needed;
- experiment protocols, table templates, and author-input checklists when real experiments or data are missing;
- a reviewer response that does not claim completed work unless changes were actually applied.

The deterministic `full` mode may produce draft scaffolds for inspection. Those scaffold outputs must not be treated as final deep revision quality unless the deep planner/auditor loop passes.

Final DOCX outputs:

- `05_修改建议版.docx` inserts highlighted suggestions and notes.
- `06_整合修改稿.docx` directly integrates actions that do not require author input and skips actions that need real data, source figures, reference verification, or author decisions.

## Adapter Rule

Do not put the only copy of an agent definition inside `.claude/agents`. Keep provider-neutral definitions in `agent_specs/agents`, then use `scripts/sync_agent_adapters.py` to refresh Claude Code adapter files.
